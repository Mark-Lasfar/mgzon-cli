# Privacy Policy

Last Updated: December 2024

## Overview

MGZON CLI is committed to protecting your privacy. This Privacy Policy explains how we handle information when you use our CLI tool.

## Information We Collect

### 1. **Authentication Information**
- API keys (encrypted and stored locally)
- User identifiers (for service access)
- Session tokens (temporary)

### 2. **Configuration Information**
- Project settings (stored in .mgzon.json)
- User preferences (stored in ~/.mgzon/config.json)
- Environment configurations

### 3. **Usage Information** (Optional)
- Anonymous usage statistics (opt-in only)
- Error reports (with user consent)
- Performance metrics

## How We Use Information

### Service Operation
- Authenticate API requests
- Manage user sessions
- Store user preferences
- Provide CLI functionality

### Improvement
- Fix bugs and errors
- Improve performance
- Enhance user experience
- Develop new features

## Data Storage

### Local Storage
- All sensitive data stored locally
- Encrypted configuration files
- User-controlled data location

### No Cloud Storage
- We do not store your data on our servers
- No data synchronization
- Complete user control

## Data Sharing

### We Do Not Share
- We do not sell your data
- We do not share with third parties
- We do not use for advertising

### Legal Requirements
We may disclose information if required by law or to protect our rights.

## Your Rights

### Access and Control
- View your stored data
- Delete your data at any time
- Export your configuration
- Opt-out of data collection

### Security
- Use strong passwords
- Protect your API keys
- Regular security updates
- Secure your devices

## Security Measures

### Protection
- Encryption of sensitive data
- Secure authentication
- Regular security audits
- Vulnerability scanning

### Best Practices
- Follow security guidelines
- Use environment variables
- Regular key rotation
- Secure file permissions

## Changes to This Policy

We may update this policy periodically. We will notify users of significant changes through:
- GitHub releases
- Documentation updates
- Version notifications
